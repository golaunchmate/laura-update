The Diagnosis

What was wrong with v1

I ran LaunchMate through its own Neal Test tool and realized: it felt like Mad Libs.

Every scenario followed the same pattern:

Technical Solution

Move creativity from code to agent

Old architecture: Python function with if/else logic generates scenarios

def run_neal_test(): if keyword in description: return template.fill(role, pricing) # Hardcoded patterns = boring

New architecture: Tool returns framework, agent generates creative scenarios

def run_neal_test(): return """ AGENT: Generate 5 unique scenarios. Make each one creative, specific, and exciting. Analyze their startup deeply. No templates. """

Key insight: Tools provide structure. Agents provide creativity.

The Impact

Why creativity beats templates

Old tool reaction: "Yeah, I could text some people. But I probably won't."

New tool reaction: "Wait, I could actually DO that? That's wild. Let me try."

The goal of Neal Test isn't just to provide instructions. It's to spark action.

If a scenario doesn't excite you, you won't execute it.

If you don't execute it, you don't learn.

If you don't learn, you build the wrong thing.

Creative scenarios ‚Üí excitement ‚Üí execution ‚Üí learning ‚Üí better product

Templated scenarios ‚Üí boredom ‚Üí procrastination ‚Üí building without validation

The Principle

Tools provide structure, agents provide creativity

"Don't hardcode creativity. Hardcode the framework that enables it."

This pattern works for more than Neal Test:

  • Tidbits: Tool provides HTML structure, agent writes unique narrative
  • Check-ins: Tool provides momentum framework, agent provides personalized coaching
  • OCF: Tool provides agent creation infrastructure, agent designs unique capabilities

The architecture decision: Move creativity OUT of tools, INTO agent reasoning.
