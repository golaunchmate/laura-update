The Tool

What create_tidbit does

The create_tidbit tool generates complete, scroll-snap HTML pages following LaunchMate's narrative format.

It takes structured input (title, subtitle, date, pages) and outputs a production-ready HTML file with:

  • Dark mode styling
  • Inter + JetBrains Mono fonts
  • Scroll-snap navigation
  • Card-based layout
  • Color-coded highlights (blue/green/orange)
  • Interactive JavaScript features

The Pipeline

How the workflow works

Step 1: You call create_tidbit with structured data (title, subtitle, date, array of pages)

Step 2: Tool generates complete HTML with all styling, fonts, and JavaScript baked in

Step 3: Tool returns the full HTML as a string

Step 4: You deploy using update_github_pages_html to push to GitHub

The Architecture

What's inside the tool

The create_tidbit tool contains:

1. Complete HTML template with embedded CSS and JavaScript

2. LaunchMate narrative styling:

  • Color system: blue for highlights, green for decisions, orange for tensions
  • Typography: Inter for body, JetBrains Mono for code/labels
  • Dark mode (#0f0f0f background, #1a1a1a cards)

3. Interactive features:

  • Keyboard navigation (arrow keys, spacebar)
  • Scroll-snap for smooth page transitions
  • Responsive design (mobile-friendly)

Everything self-contained. No external dependencies except Google Fonts.

Usage Guide

How to use create_tidbit

Required parameters:

  • title: Main tidbit title
  • subtitle: Brief description
  • date: Date string (e.g., "January 2026")
  • pages: Array of page objects

Page object structure:

{\n "section_label": "Small uppercase label",\n "page_title": "Page heading",\n "content": "HTML content here",\n "is_final": false // true for last page\n}

Content accepts full HTML: Use span classes for colors, divs for structure, code blocks, lists, etc.

Design Principle

Why this architecture works

Separation of concerns:

  • create_tidbit: Generates beautiful HTML
  • update_github_pages_html: Handles deployment
  • Agent: Provides content and narrative

Each piece does one thing well. Tools provide structure, agents provide creativity.

The pattern: Don't build monolithic tools that try to do everything. Build composable tools that chain together.

create_tidbit doesn't know about GitHub. update_github_pages_html doesn't know about tidbit formatting. Clean interfaces.
