v1 Architecture

How the old tool worked

v1 Used hardcoded if/else logic to generate scenarios:

# Simplified v1 logic if "accountability" in description: role = "accountability coach" routine = "Text at 8am, process at 12pm, feedback at 6pm" pricing = "$40/week" elif "coordinate" in description: role = "personal coordinator" routine = "Collect updates, compile manually, distribute to team" pricing = "$35/week" # Fill template with variables return template.format(role, routine, pricing)

Problem: Every scenario for "accountability" startups looked identical. Same structure, same pricing pattern, same daily routine.

Result: Boring. Founders read it and think "yeah, I could do that" but don't get excited.

2/6

v2 Architecture

How the new tool works

v2 Tool returns a framework, not scenarios:

# v2 logic def run_neal_test_session(startup, product): return f""" NEAL TEST FRAMEWORK FOR: {startup} CORE REQUIREMENTS: 1. Manual Service Delivery (no tech) 2. Customer Payment (they must pay) AGENT: Generate 5 unique scenarios. - Creative and non-obvious - Specific to their context - Exciting enough to inspire action - Vary approach, pricing, delivery [Examples of good vs bad scenarios] Framework returned. Agent generates creative scenarios in response. """

Process: Agent reads framework ‚Üí analyzes startup deeply ‚Üí generates 5 completely unique scenarios tailored to their specific context
