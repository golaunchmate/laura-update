Architecture Pattern

Instructions in Tools vs Memory Blocks

Where should agent instructions live?

January 2026 // scroll to read

The Confusion

Where do instructions go?

I was building the create_tidbit tool and realized: the formatting rules could live in two different places.

Option 1: A memory block called <narrative_format> that agents read.

Option 2: Inside the tool's docstring itself.

But which one makes sense? And what's the actual difference?

1/7

Option 1

Memory Block Approach

This is what we do now with <narrative_format>.

# Memory block in agent's context <narrative_format> TONE: Engineer walking through process DESIGN: Scroll-snap, dark mode COLOR CODING: - Highlights: #60a5fa (blue) - Decisions: #34d399 (green) </narrative_format>

The agent reads this memory block before calling any tool.

Instructions are shared across all agents that have this memory block attached.

2/7

Option 2

Tool Docstring Approach

Instructions live inside the tool itself as a docstring.

def create_tidbit(...): """ Creates tidbit HTML. TONE: Engineer walking through process COLOR CODING: - Highlights: #60a5fa (blue) - Decisions: #34d399 (green) """ # Tool code here

The agent reads the docstring when it considers using the tool.

Instructions travel with the tool automatically.

4/7

Real Example

The create_tidbit tool

When I created create_tidbit(), I put the instructions in the docstring.

Why? Because:

  • The formatting rules are specific to this tool
  • Any agent that attaches the tool gets the instructions automatically
  • No need for a separate narrative_format memory block

The tool is self-documenting ‚Äî it knows how to use itself.

5/7

Decision Framework

When to use each approach

Use memory blocks when:

  • Instructions apply to multiple tools
  • Users need to customize the behavior
  • Instructions change frequently
  • You want shared context across agents

Use tool docstrings when:

  • Instructions are tool-specific
  • You want portability (tool = instructions)
  • Format is stable/standardized
  • You want one source of truth
