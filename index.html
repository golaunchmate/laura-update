The Issue: When scaling erica-summarizer from 10 teams to 19 teams, the HTML email reports hit Gmail's size limits (~102KB), causing truncation and breaking the entire system.

Failed Approaches:

  • ‚ùå Multi-part emails: Buggy in Letta, creates email flood

  • ‚ùå Drastically shorten content: Loses detail Erica needs for interventions

‚úÖ The Breakthrough Solution

üéØGitHub Pages + Automated Deployment

The Winning Approach: Instead of sending full HTML via email (which truncates), automatically deploy the HTML report to GitHub Pages and send Erica a short email with a link.

How It Works:

  1. Erica-summarizer generates full HTML report (unlimited size)
  2. Agent calls update_github_pages_html tool
  3. Tool commits HTML to GitHub repo via API
  4. GitHub Pages automatically deploys updated report
  5. Erica receives simple email with link to live report

üí° Key Learnings & Insights

1. Scale Reveals System Limitations

The erica-summarizer worked perfectly at 10 teams but broke at 19 teams. This taught us to test at realistic scale early, not just proof-of-concept scale.

2. External Infrastructure > Fighting Platform Limits

Instead of fighting email size limits with multi-part messages and truncation workarounds, we leveraged external infrastructure (GitHub Pages) that doesn't have those constraints.

3. Automation Stack Integration

The real power comes from connecting multiple systems: Letta agents ‚Üí Custom tools ‚Üí GitHub API ‚Üí GitHub Pages ‚Üí Email notification. This creates a fully automated pipeline.

4. User Feedback Drives Architecture

Erica's specific feedback (focus on check-ins, shorter feedback, team-level not individual) shaped the entire summarizer design. User validation isn't just for product features‚Äîit's for infrastructure decisions too.
