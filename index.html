ğŸ” Authentication: GitHub Personal Access Token

What is it?

A GitHub Personal Access Token (PAT) is a secure credential that allows programmatic access to GitHub without username/password.

How to create one:

  1. Go to GitHub â†’ Settings â†’ Developer settings â†’ Personal access tokens â†’ Tokens (classic)
  2. Generate new token
  3. Select scopes: repo (full control of private repositories)
  4. Copy token (it's shown only once!)

Security note:

  • Store in agent memory (your tools already have it)
  • Never commit tokens to public repos
  • Treat like a password

ğŸ“¡ How the GitHub API Commit Works

The Magic: Creating Commits Without Git

Traditional workflow:

git add index.html
git commit -m "Update content"
git push origin main

Our workflow: Single API call that does all of this server-side.

GitHub API Endpoint Used:

PUT https://api.github.com/repos/{owner}/{repo}/contents/{path}

What happens behind the scenes:

Step 1: Get current file SHA (if file exists)

GET /repos/golaunchmate/laura-update/contents/index.html

Response:
{
  "sha": "abc123...",  # Current file version identifier
  "content": "base64encodedcontent"
}

Why? GitHub needs the SHA to verify you're updating the correct version (prevents conflicts).

Step 2: Commit new content

PUT /repos/golaunchmate/laura-update/contents/index.html
Headers:
  Authorization: token [REDACTED]
  Accept: application/vnd.github+json

Body:
{
  "message": "Update Laura's progress",  # Commit message
  "content": "base64_encoded_html",       # New file content
  "sha": "abc123..."                      # Previous version SHA
}

What GitHub does:

  1. Decodes base64 content
  2. Creates new git commit with your message
  3. Updates file in repository
  4. Returns new commit SHA

Step 3: GitHub Pages auto-deployment

Trigger: Any commit to the repository automatically triggers GitHub Pages rebuild.

Behind the scenes:

Commit detected â†’ GitHub Actions workflow triggered 
â†’ Jekyll/static site builder runs 
â†’ Outputs to gh-pages branch or docs folder
â†’ Deployed to CDN
â†’ Live at https://golaunchmate.github.io/laura-update

Pseudo-code for update_github_pages_html:

def update_github_pages_html(html_content, github_token, repo_owner, repo_name, 
                              file_path="index.html", commit_message="Update"):
    
    # 1. Set up GitHub API client
    headers = {
        "Authorization": f"token {github_token}",
        "Accept": "application/vnd.github+json",
        "User-Agent": "Letta-Agent"
    }
    
    base_url = f"https://api.github.com/repos/{repo_owner}/{repo_name}"
    file_url = f"{base_url}/contents/{file_path}"
    
    # 2. Try to get existing file SHA (needed for updates)
    try:
        response = requests.get(file_url, headers=headers)
        if response.status_code == 200:
            current_sha = response.json()["sha"]
        else:
            current_sha = None  # File doesn't exist yet (first commit)
    except:
        current_sha = None
    
    # 3. Encode HTML content to base64 (GitHub API requirement)
    import base64
    encoded_content = base64.b64encode(html_content.encode('utf-8')).decode('utf-8')
    
    # 4. Prepare commit payload
    payload = {
        "message": commit_message,
        "content": encoded_content,
        "branch": "main"  # or "master" depending on repo
    }
    
    # Include SHA if file exists (update) vs create new
    if current_sha:
        payload["sha"] = current_sha
    
    # 5. Make API request to commit file
    response = requests.put(file_url, headers=headers, json=payload)
    
    # 6. Handle response
    if response.status_code in [200, 201]:
        return {
            "status": "OK",
            "message": f"âœ… Successfully updated {file_path}! View at: https://{repo_owner}.github.io/{repo_name}",
            "commit_sha": response.json()["commit"]["sha"]
        }
    else:
        return {
            "status": "ERROR",
            "message": f"Failed to update: {response.json()}"
        }

ğŸ” Debugging & Troubleshooting

Common Issues:

âŒ "404 Not Found" after deployment

  • Cause: GitHub Pages not enabled yet
  • Fix: Go to repo Settings â†’ Pages â†’ Enable from main branch

âŒ "Permission denied" error

  • Cause: Token doesn't have repo scope
  • Fix: Regenerate token with full repo permissions

âŒ "File SHA mismatch"

  • Cause: Someone else updated file between read/write
  • Fix: Tool automatically re-fetches SHA and retries

âŒ "Content too large"

  • Cause: GitHub API has 1MB limit per file
  • Fix: Split into multiple files or compress images

âŒ Old content still showing

  • Cause: Browser cache or GitHub Pages delay
  • Fix: Hard refresh (Ctrl+Shift+R) or wait 2 minutes
